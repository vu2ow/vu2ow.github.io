Renmeteor program

This program provides an easy UI for generating ini files for lrpt decoder and meteorgis program.

0. Receive M-N2 and record the QPSK demodulator output to a folder.

1. Create a folder in drive C called Meteor

2. Copy M2_LRPT_Decoder.exe there. I am using version 2015.7.1.0033

3. Create a folder called "images". This is the folder where LRPT decoder will create its files and logs (Raw images)

4. Create a folder called meteorgis and copy meteorgis files into it

5. Create a folder called newimages for meteorgis program to create its output

6. Copy renmeteor.exe to any of the folders above

7. Run renmeteor.exe and follow the steps on the screen. LRPT decoder can be run in two modes - offline and online via TCP IP, which meteorgis can read directly. I have been running it in offline mode only. All the procedures outlined here work will if the above instructions are followed strictly. Step 6 helps to rename the files generated by lrpt decoder as required by meteorgis. Without this, meteorgis will not run.

;If you use it in automatic mode with SDRSharp & the plugin Meteor Demodulator, you normaly don't have to specify any option.
;"/nolive" By default MeteorGIS will work in automatic mode. It can launch the decoder (M2_LRPT_Decoder.exe) after creating the right ini file for it.

; If you didn't use this option (mean automatic mode) take care to not specify an input file (but folder instead)
